# genai-trend-sync - 初期要件定義（アイデアメモ）

## このドキュメントの位置づけ

このドキュメントは壁打ち・ブレストの成果物で、正式な仕様書ではありません。
`/setup-project` 実行時に自動的に読み込まれ、PRD等の正式ドキュメント作成のインプットになります。

## プロダクトビジョン
**「読むのが楽しみになる、自分専用のAIトレンド・キュレーター」**
Anthropic、OpenAI、Google（Gemini）などの主要AIサービスの激しいアップデート情報に振り回されるのではなく、システムが自動で収集し、自分が「読みたい・頭に入りやすい」形式に編集してから届けてくれる環境を作る。

## 解決したい課題

1. **生成AIに関連する情報のアップデート頻度と量についていけない**
   - 巨大テック企業のAIモデル更新や新機能発表が多すぎ、タイムリーにインプットするのが困難。
2. **「単なる要約」はつまらないし、記憶に残らない**
   - 従来の要約ツールやRSSリーダーでは、箇条書きの無味乾燥なテキストが出力されるだけで、読書体験として面白くなく、自身の知識として定着しない。

## コア機能（MVPスコープ）

### 1. 情報の自動収集
- 主要なAI企業（OpenAI, Anthropic, Googleなど）のアップデート情報を収集する。
- *(※MVP段階では、公式ブログのRSSなどを固定のソースとして扱う)*

### 2. LLMによる「読みやすい形式」への再構成と、画像生成プロンプトの作成
- 単純な要約ではなく、ユーザーの興味を引き、頭に入りやすい形式（ストーリー仕立て、技術的インパクトの解説など）に記事を再構成する。
- 同時に、その記事のコア概念を視覚的に表現するための「画像生成用プロンプト」をLLMに作成させる。

### 3. イメージ図の自動生成（コア価値の強化）
- ステップ2で生成したプロンプトを画像生成APIに渡し、記事の理解を助けるアイキャッチ画像や概念図（イメージ図）を自動生成する。

### 4. Notionへの自動データベース投稿
- 再構成されたテキストと生成された画像を組み合わせて、Notionのデータベースに1つのリッチな記事として自動で保存（投稿）する。
- これにより、Notionを自分専用のナレッジベースとして活用できる。

## ターゲットユーザー
自分自身と社内のテクノロジー部門への情報展開

## アーキテクチャと実行環境（MVP）

- **実行環境**: GitHub Actions
- **トリガー**: 週に1回の定期実行（Cronスケジュール）
- **バックエンド**: Node.js (TypeScript)
- **データ処理LLM**: Gemini 2.5 Pro ※geminiAPIの最新モデル（推論）、Nano Bananaの最新モデル
- **出力先**: Notion API を使用したデータベースへの書き込み
